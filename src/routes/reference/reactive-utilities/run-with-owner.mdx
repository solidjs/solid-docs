`runWithOwner` executes a function under the provided owner.

## Import

```ts
import { runWithOwner } from "solid-js";
```

## Type

```ts
type Owner = unknown;

function runWithOwner<T>(owner: Owner | null, fn: () => T): T | undefined;
```

## Parameters

### `owner`

- **Type:** `Owner | null`
- **Required:** Yes

Owner used while executing `fn`.

### `fn`

- **Type:** `() => T`
- **Required:** Yes

Function executed under `owner`.

## Return value

- **Type:** `T | undefined`

Returns the value produced by `fn`.

## Behavior

- Computations created inside `fn` are attached to the provided owner.
- Context lookup inside `fn` walks the provided owner tree.
- Ownership affects cleanup and context resolution.
- Ownership does not restore dependency tracking across asynchronous boundaries.
- Code after the first `await` in an `async` function runs without reactive dependency tracking.

## Examples

### Restore an owner in a callback

```ts
import { createEffect, getOwner, runWithOwner, useContext } from "solid-js";

const owner = getOwner();

setTimeout(() => {
	runWithOwner(owner, () => {
		const value = useContext(FooContext);

		createEffect(() => {
			console.log(value);
		});
	});
}, 1000);
```
