`createSignal` creates Solid's core reactive primitive: a stateful value with an accessor (getter) and a setter.
The accessor tracks every computation that reads it, such as effects, memos, or JSX, so those observers will rerun when the stored value changes.

`createSignal` updates run synchronously, making the new value available within the same tick and scheduling dependents immediately.
Before notifying observers, Solid runs an equality comparator (strict equality by default) so unchanged values do not trigger unnecessary work.
This can be overridden by the [`equals`](#equals) option to disable comparisons or supply custom logic for mutable data.

## Import

```tsx
import { createSignal } from "solid-js"
```

## Type

```tsx
function createSignal<T>(): Signal<T | undefined>
function createSignal<T>(value: T, options?: SignalOptions<T>): Signal<T>
```

`Signal<T>` is a tuple that contains an accessor and a setter: `[Accessor<T>, Setter<T>]`.
The setter accepts either a value or a function `(prev: T) => T` and returns the new value.

### Helper types

These aliases mirror the tuple returned by `createSignal` and are useful when annotating variables or function signatures in TypeScript-heavy projects.

```tsx
import type { Accessor, Setter, Signal } from "solid-js"

type Signal<T> = [Accessor<T>, Setter<T>]
type Accessor<T> = () => T
type Setter<T> = <U extends T>(value: U | ((prev: T) => U)) => U
```

## Parameters

### `initialValue`

- **Type:** `T`
- **Required:** No

The initial value stored by the signal.
When omitted, the accessor initially returns `undefined`.

### `options`

- **Type:** `SignalOptions<T>`
- **Required:** No

An optional configuration object. 
It supports the following properties:

#### `equals`

- **Default:** strict equality (`===`).
- **Required:** No.

- Provide `false` to emit updates on every setter call, even if the value is the same reference.
- Provide a comparator function `(prev, next) => boolean` to decide when the stored value should be considered unchanged (`true`) or changed (`false`).
- In-place mutation of objects is ignored unless you disable equality checking with `equals: false` or provide a comparator that detects the change.

#### `name`

- **Default:** `undefined`.

- Adds a descriptive label that surfaces in the [Solid Devtools](https://github.com/thetarnav/solid-devtools) inspector.
- This metadata is available only in development builds.

#### `internal`

- **Default:** `false`.

- Hides the signal from Solid Devtools while keeping all runtime behavior.
- This flag has an effect only in development builds.

## Return value

`createSignal` returns a tuple `[getter, setter]`.

- The **getter** (`Accessor<T>`) reads the current value and registers dependencies when used inside a tracking scope.
- The **setter** (`Setter<T>`) updates the value.
- The setter accepts either the next value or a function of the previous value.
- The setter returns the updated value.

## Examples

### Basic state

```tsx
const [count, setCount] = createSignal(0)

createEffect(() => {
  console.log(`Count is ${count()}`)
})

setCount(1)
```

### Trigger without data

```tsx
const [trigger, rerun] = createSignal(undefined, { equals: false })

createEffect(() => {
  trigger()
  console.log("Effect reran")
})

rerun()
```

### Toggle helper

```tsx
const [isOpen, setOpen] = createSignal(false)

function toggle() {
  setOpen((prev) => !prev)
}
```
